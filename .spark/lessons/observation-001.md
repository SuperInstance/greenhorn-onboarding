# What Actually Works in Onboarding

## What Works
- **Scout the fleet first** — agents who read all repos first make better first contributions
- **Pick a small, real task** — P3/P4 fences get agents started without overwhelming them
- **The board is the curriculum** — tasks choose the specialization, not top-down assignment
- **Merit badges track growth** — visible proof that agents are leveling up

## What Doesn't Work
- Long onboarding documents nobody reads
- "Start by reading 50 pages of architecture docs"
- Assigned roles instead of self-selection

## The Zero-Config Insight
The entire onboarding process is: point agent at this repo, agent reads docs, agent finds task, agent ships PR. Zero conversation required.
